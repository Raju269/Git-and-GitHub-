VERSION CONTROL SYSTEMS

Version control enables organized code management and collaboration, with
Centralized Version Control (CVCS) and Distributed Version Control (DVCS)
being the two main types, chosen based on the nature and scale of the project.

────────────────────────────────────────
CENTRALIZED VERSION CONTROL (CVCS)
────────────────────────────────────────

Centralized Version Control (CVCS) is a system where there is a central
repository that stores all the code, and developers access this repository
to make changes.

Examples: SVN, CVS

FEATURES:
• Single Central Repository  
• Centralized Version History  
• Real-Time Collaboration  
• Simple Setup  

USE CASES:
• Small teams  
• Projects needing real-time access  
• Enterprises with strict access control  

────────────────────────────────────────
DISTRIBUTED VERSION CONTROL (DVCS)
────────────────────────────────────────

Distributed Version Control (DVCS) allows every developer to have a complete
local copy of the repository, including full history.

Examples: Git, Mercurial, Bazaar

FEATURES:
• Full Local Repository  
• Easy Branching and Merging  
• No Single Point of Failure  
• Offline Work Support  

USE CASES:
• Large & distributed teams  
• Open-source projects  
• Frequent experimentation  

────────────────────────────────────────
CVCS vs DVCS COMPARISON
────────────────────────────────────────

CVCS:
• Single central repository  
• Requires server to commit  
• Slower branching & merging  
• Central server is a failure point  

DVCS:
• Full copy per developer  
• Offline commits allowed  
• Fast branching & merging  
• Every repo acts as backup
